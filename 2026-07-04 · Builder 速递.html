今日主线

核心 builder 今日速览

  • Fable 5 回归成长周末焦点：一线 builder 集中输出用法、token 消耗观察和"是不是被削"的澄清。
  • "context / applied 层"被反复点名为 AI 竞争的真正战场：谁能组织领域知识、路由模型、端到端解决业务问题，谁就有护城河。
  • Agent 基建往"自省 / 可观测 / 沙箱"演进：让 agent 复盘过去的运行、生成新 skill，以及在更强的沙箱里跑真实运行时。

今天这批 builder 的重心明显收在两处：一是 Fable 5 长周末回归带来的密集实测——用法（Thariq、Peter Yang）、token 量级（Dan Shipper）、以及对"是不是被削"的路由/fallback 澄清（Dan Shipper），提醒大家跑分争议常来自路由而非模型本身。二是一条更硬的结构性判断正在成形：竞争的真正战场是 context 与 applied 层——Levie 给出完整框架、Peter Yang 用"超级 app"具体化、Nikunj 从单 key 多模态的开发者体感佐证，共识是"谁能组织领域知识、跨模型路由、端到端解决业务问题，谁的护城河最深"。与此并行，Rauch 把 agent 基建推向"更强沙箱 + 自省式可观测"。对 builder 的信号很清楚：模型能力已被默认，接下来的差异化在于你能给 agent 喂多好的 context、搭多可靠的运行/观测底座，以及把它嵌进真实工作流的能力。

三栏速览 · 一句话
  • 在讨论什么Fable 5 的限时回归点燃了全圈的"7/7 前冲刺"，从独立开发者到 Anthropic 自家 PM 都在算同一笔 token 账、赶同一个截止日。而紧随其后的"它是不是被偷偷 reroute 削弱了"之争，把这场狂欢里的一丝不信任摆上了台面。
  • 什么在变Agent 正在离开 localhost、搬进随开随用的远程沙箱——"合上笔记本、去晒太阳、手机上 merge"从段子变成了实际工作流。与此同步的是一种新默认姿势：让最强的模型去指挥更便宜的模型干活。
  • 从业者最在意什么token 预算和 rate limit 已经硬生生变成一线的选型约束——用什么工具、留不留旧工具，越来越多是被额度而非能力决定的。与此同时，"人到底还需不需要顾问 / FDE"这个更尴尬的问题也被摆上了桌面。
PART 1 · 逐账号

我们跟踪的核心 builder

Aaron Levie（Box CEO）

Levie 认为 AI 的核心竞争是"抢 context"。agent 好不好用，取决于它有没有领域知识、有没有接到正确的 context 和工具、以及能否嵌进用户可审查可介入的工作流。由此推论：能在自己 agent 里沉淀最多、最优质 context 并为任务选对模型的平台，才是 agent 干活最好的地方（编程、法务、客服 agent 都已印证）。他借此反驳"applied 层只是 LLM wrapper"——把关键知识按权限治理好、持续为 agent 改进 context，才是价值所在。未来这层会在模型间路由：前沿模型做规划/编排/审查，大量中间工作交给低成本开/闭源模型；applied 层甚至可以拿近前沿基座做领域后训练，并靠 FDE 做变更管理，端到端解决问题者护城河最深。

为什么重要 给出了"应用层 vs 模型层"价值归属的一套完整逻辑，是理解 agent 商业化往哪走的框架。
原文链接：@levie
★ 1
Peter Yang（产品/评测）

Yang 抛出 "AI 超级 app 时代" 判断：Anthropic 与 OpenAI 都想用 agent 重做所有知识工作，从编程扩展到设计、金融、法律、科研——他称知识工作者采用 Codex 的速度是开发者的 3 倍。超级 app 之争主角是 Codex、Claude、Cursor 等，配合 browser/computer use，Codex 已能替他处理电脑上几乎所有事；OpenAI 靠 ChatGPT 8 亿+ 周活占位有利，可能把 ChatGPT 与 Codex 并成一个 app。他直言 Anthropic 企业侧领先但产品线太碎（Claude、Cowork、Claude Code、Claude Design 不该各自为政，应像"一个能干的同事"）。实操上他分享了 Fable 用法（先用便宜模型备 context、用 Fable 规划再换模型执行、用 Medium 低 effort 并盯着它做）和一条 Codex 工作流（ffmpeg 切视频片段 + browser use 操作 Meta Business Suite/TikTok UI 自动排期，20+ 步至今没塌）。

为什么重要 把"超级 app 竞争"和各家产品形态的优劣讲得很具体，也给了 agent 编排的落地手法。

PART 2 · 主题重构

整个 builder 圈：在讨论什么 · 什么在变 · 最在意什么

今日气象

这个长周末，整个 builder 圈被同一件事牵着走：Anthropic 的顶配模型 Fable 5 限时回归，只"包含"到 7 月 7 日，之后转为全家最贵的按量计费。于是全圈进入一种节庆式的赶工——一边狂欢式地"趁免费把最贵的活干完"，一边精打细算地盯着 token 和限额。比起模型本身，大家更焦虑的是"还能烧多久"。而在这层喧闹之下，一条更结构性的暗流正在成形：agent 正悄悄搬离本地笔记本、住进远程沙箱，并开始用顶级模型去调度更便宜的模型干活。

01

在讨论什么

Fable 回归的第一反应不是惊叹性能，而是"限时"带来的赶工与 token 焦虑 —— 从 Theo 清空积压、Cat Wu 征集周末 demo，到 Dan Shipper 逐条给任务标价、Greg Isenberg 发"最后期限"科普，好几拨互不相干的人这几天都围着同一个 7/7 大限打转。Nikunj 甚至打趣，实验室专挑长周末放模型，就是要"给大众时间去把玩、被震撼，然后陷得更深、token 焦虑更重"。

我把等 Fable 期间攒下的活儿全清了

"一天干完，现在不知道该干嘛了"。 —— Theo Browne（独立开发 / T3）@theo

Fable 像家里的好瓷器：只包含到 7/7，之后 $10/$50 每百万 token、吃额度约为 Opus 的两倍、最多占掉周限额的一半

"这周末就该把最贵的大活先干掉"。 —— Greg Isenberg（独立开发 / 创业）@gregisenberg

"Fable 是不是被削弱了"成了一场没有共识的口水战 —— 起因是一句"Fable 5 不是被削、是被屠杀了，问题出在硬性护栏"的断言，随即分裂成两派：Dan Shipper 说"其实是同一个模型，只是更容易回退到 Opus，skill issue"，Theo 也称自己"一次都没被 reroute 过"；但评论区并不买账。真正的分歧不在模型，而在厂商的暗箱——有人晒图称一整段会话 70% 被路由到 Opus、额度照扣，还有人直接质疑大 V 账号被"优待"。

那个基准测试恰好踩中了会触发 reroute 的点，"Fable 表现没变，只是更容易被转走"。（评论：我每次都被转，这次 70% 路由到 Opus，额度没了）

Theo Browne（独立开发 / T3）@theo
02

什么在变

写代码这件事正在整个搬离本地机器 —— Sourcegraph 的 Beyang 和 Vercel 的 Rauch 这两条互不相干的线，本周撞到了同一个方向：把 agent 跑在远程沙箱里，本地只剩一块观察屏。Beyang 反复强调"改这个 diff 没碰过编辑器和终端，闭环正在收拢"；Rauch 则把 Vercel Sandbox 打通到能无约束跑 docker 和 fuse、给 agent 挂 S3 文件系统。重活挪到云端，笔记本退化成遥控器。

这些都不在 localhost 上跑

"开一个 orb，让它截图自证，合上电脑去晒太阳，手机上点合并"。 —— Beyang Liu（Amp / Sourcegraph）@beyang

Sandbox 现在能无约束跑 docker 和 fuse、给 agent 挂 S3 文件系统。（高赞评论：所谓"agent 需要专用基建"的论调一再坍缩成"agent 只需要 unix"

那套 50 年的老抽象本来就是 agent-native 的） —— Guillermo Rauch（Vercel CEO）@rauchg

"一个巨无霸模型包打天下"正在让位给多模型编排 —— Simon Willison、Beyang、Theo、Peter Yang 从各自角度不约而同地在做同一件事：让顶级模型自己判断、把活派给更便宜的 subagent 或另一个厂商的模型。Simon 的那条省 token 心法转了几千赞，评论区一串人补充自己也让 Claude 去调度 Codex、在 CLAUDE.md 里写"为每个子任务挑合适的模型"。未来更像多 agent 的组合智能，而非单一庞然大物。

"所有编码任务，你自己判断挑一个合适的低配模型、放进 subagent 去跑"

似乎省了很多 token。 —— Simon Willison（独立开发 / 教育）@simonw

GLM 5.2 当主驱动不错，但配上专用 subagent 才真正惊艳

"前沿越分化，你会看到越多组合式智能，而不是一个单体大模型"。 —— Beyang Liu（Amp / Sourcegraph）@beyang

前端底座悄悄换代：shadcn/ui 把默认组件库从 Radix 换成 Base UI —— 这不是一次普通版本更新，而是社区用脚投票的结果（shadcn/create 新项目里 Base UI 以 2:1 压过 Radix）。更值得注意的是迁移方式本身就很"agent 时代"：官方给的不是文档，而是一个让 coding agent 逐个组件平滑迁移的 skill。

从今天起 Base UI 成为 shadcn/ui 默认库；迁移只需 `npx skills add shadcn/ui`，再让 agent"把 alert-dialog 迁到 base ui"，一次一个、项目始终可运行。

shadcn（Vercel / shadcn/ui）@shadcn
03

从业者最在意什么

rate limit 精贵到要"省着用"，反而逼出了多工具并存的现实 —— jxnlco 抛出"既然有 Codex，你还留着 ChatGPT 干嘛"，五百多条回复几乎拼出了一线的真实用法：不舍得拿 Codex 的额度去回封邮件、某些能力（5.5 Pro、Deep Research）只在 ChatGPT 有、日常提问不想被一堆 MCP 和 skill 文件把 context 撑爆。决定用哪个工具的，往往是额度而不是能力。

"既然用了 Codex，还有什么理由继续用 ChatGPT？"（最高赞回复：我才不会为了回封邮件烧掉 Codex 的 rate limit）

Jason Liu（独立开发 / 教育）@jxnlco

"AI 让你一人顶二十人"与"我们砸几十亿派驻工程师帮你落地"两套叙事，被当面戳穿矛盾 —— Hamel Husain 直言这两句话根本不自洽，并追问：FDE 撤走之后 AI 还转得动吗，还是会对顾问产生终身依赖，而顾问又有什么动力让自己被淘汰？有意思的是 Aaron Levie 几乎同一天给出了对立答案——他认为正是这层"应用层 + FDE"最能沉淀 context、做变更管理，因而护城河最深。

两套 AI 叙事互相打架：一边"学会用 Claude 就能一人顶二十人"，一边"太难了所以我们砸几十亿派 FDE 帮你实施"

"我更信前者"。 —— Hamel Husain（实践 / 教育）@HamelHusain

AI 的胜负手是"抢占 context"：能组织并治理好领域知识、并在前沿模型做规划与便宜模型做量产之间路由的应用层，护城河最深，而这正是 FDE 的价值所在。

Aaron Levie（Box CEO）@levie

想让 agent 干得好，改造的其实是你的 codebase 和流程，而不是换个更强的工具 —— Beyang 借一篇"41 个 AGENTS.md 文件 + orb 配置 + dev server 工具"的实践点破这点；Quinn Slack 则给了个具体招数：让 agent 主动埋"面包屑"，在需要日后清理的地方撒 TODO 注释，既方便后续批量清理，也能警告别的 agent 别继续沿用旧写法。

"用 agent 干活，七分在你的代码结构和流程，三分才在工具本身。"

Beyang Liu（Amp / Sourcegraph）@beyang

让 agent 留面包屑：需要维持几天向后兼容时，就让它到处撒 `TODO(sqs)` 注释，方便日后（甚至每日）自动清理。
